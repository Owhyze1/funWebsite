React Notes

React

JavaScript library for building fast and interative user interfaces.

Created by FaceBook in 2011.

Most popular library for building user interfaces

React Components

An entire React application can be modelled as a set of independent, isolated, and reusable components, used to design complex layouts.

Websites can be split into smaller and smaller components

The App component

Every React app has at least one component referred to as the App component

The App component represents the React app and all other components are placed in it

This gives it a tree-like component hierarchy

Summary

React is a library, which means it has a very specific purpose. It defines user interfaces, also called Views.

It can be used for part of your application, because it only handles the views. Contrast this with Angular, which has built-in support for network requests, etc.

Single Page Applications (SPA) and Multiple Page Applications (MPA)

Single Page Application (SPA)

App that works in the browser without requiring page reload during use.

A sole HTML page that loads all other content using JavaScript

Examples: FaceBook, GMail, Instagram, Trello

Multi-Page Application (MPA)

Every change sends request to the server to render a new page in the browser

More "traditional" Websites

React can be used for both SPAs and MPAs, but is mostly used for SPAs.

Introduction to JSX

JSX

A syntax extension in JavaScript that defines React elements with a syntax very similar to HTML

It defines the look of the UI, i.e., the structure of the React Components.

JSX combines rendering logic with other UI locgic (e.g. event handling, state changing, data displaying, etc)

Babel

JSX code gets converted to JavaScript by Babel

It uses React.createElement( ), which is imported from the React module in the import React from 'React'

The React.createElement( ) method can have 3 arguments

      Element name
      Properties of attributes
      Child components

JSX code is optional. You can write in JavaScript, but it's more difficult to read

Custom Components

HTML elements have the first argument of the React.createElement( ) method in lowercase and in quotations. Custom elements are passed as they are written in JSX without quotations.

Styling React Elements

Two options

  1. Inline Styles
  2. External Style

Inline styles are written as an object within another object, that is the value of the style attributes. The keys are the CSS properties. Dashes are removed and camel case is used instead. The values are the CSS property values enclosed in quotation marks.

The problem with inline styles is that you cannot use a lot of CSS properties, such as media queries, pseudo classes, pseudo elements, etc.

It is adviseable NOT to use the id attribute because it does not allow for the reusability of components

Create React Components Dynamically

You can use an array and the map() function to dynamically display components

The "key" property

React uses this property to determine which items are changed, added, or removed in dynamically generatd lists

It must UNIQUELY identify the list item

It must be given to OUTERMOST elements being rendered by the list

You can add an "id" property to the object or generate it dynamically

Component

Lets you split the UI into independent, reusable pieces

  2. Class-based

Exporting Modules

  2. Name Export - export MULTIPLE modules from the same file

Named exports have to be imported under the same name. The name can be changed if it is imported as import {namedExport as newName} from './namedExportLocation';

Add a parameter to the functional component and use parameter object keys in the inside of the functional component

Intro to "CSS Modules" tool

Global styles add a dependency between different components.

To make components truly independent, use style classes scoped locally to the component.

CSS Modules create a local scope for CSS Classes. A unique class name is used for all classes in the CSS module file using the following:

  • file name
  • class name
  • random hash value

A CSS Modules compiler creates this unique name, as well as a CSS compiled file and a JavaScript file. This JavaScript object is what will be used inside the the components to add classes to React elements.

CSS Modules compiler is automatically installed when using create-react-app

Using CSS Module

  1. Create a CSS module file as fileName.module.css
  2. Add CSS properties to the module file
  3. Import the file into the appropriate JS file
  4. Pass the class name as an object using dot notation i.e. className={classes.BlogCard}

Stateful and Stateless Components

Stateful v. Stateless

Stateful Components - Components WITH state that are used when you want to add functionality

Stateless Components - Components WITHOUT state that are used only show something on the screen

State

State is similar to props, but is private and fully controlled by the component.

It is an object and can hold any type of value

State functionality comes from React's Component class

Two Major Types of Components

  • Stateful Components
  • Stateless Components

Stateful Components

Characteristics

  • Class-based or Container or Smart components
  • Have a state object
  • Keep track of changing data via state object

Stateless Components

Characteristics

  • Function-based or Presentational or Dumb components
  • NO state object
  • Print what is given to them via props or they ALWAYS render the same thing

NOTE: A stateless component can have stateful components inside it

Creating a Class Component

Use the extends keyword with the class name as class App extends Component

Import the Component clas from react as an object OR extend the class using React.Component

Do NOT use const or let keywords because all variables and functions are now properties of the class

Do NOT use a return statement. The Component gives the class access to a method called render, which triggers all JSX code or React elements. This render method is called every time a component is created or updated

Must use the this keyword when referring to any variables or functions inside the class

Using State Property

The Component class provides access to a state property that is used to manage the state of components

At any time, the state property stores the current state of the component --> this.state = { showBlogs: true }

The Component class also provides access to a method called setState. This method is used to update the state object and run the render method again. Example: this.setState({showBlogs: false})

When using the setState method, first it updates the state object then it calls the render method. Then, the render method executes all the code of the updated state

The setState method is ASYNCHRONOUS. This is a problem when you are using the current state to update the next state. To avoid any issues, setState provides access to the previous state and props so that you can use those to update the next state. Example: this.setState((prevState, prevProps) => { return {showBlogs: !prevState.showBlogs }});
