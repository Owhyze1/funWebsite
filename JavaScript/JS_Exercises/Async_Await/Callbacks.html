Callbacks and Promises

Promise

Handles asynchronous operations by deferring code execution until an asynchronous request is completed. This allows other operations to keep running without interruption.

Promises prevent extreme nesting caused with chaining GET requests.

Three States of a Promise

  1. Pending
  2. Fulfilled
  3. Rejected

Promise Syntax

const mPromise = new Promise((resolve, reject) => {

// Call resolve() when operation is complete

// Call reject() when operation fails

}

Methods: then() and catch()

then() is called when resolve( ) is executed. It receives data passed into the resolve( ) method as arguments.

catch() method is called when the reject( ) method is executed. It receives data passed into the reject( ) method as arguments.
